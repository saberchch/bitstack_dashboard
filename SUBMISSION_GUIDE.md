# Bitstacks D-Library Submission Guide

Welcome to the Bitstacks Digital Library! This guide outlines how to prepare, format, price, and publish your academic resources to maximize peer-verification rates and BTS token yields.

---

## 1. Preparing Your Content

To maintain the high academic standard of the Bitstacks repository, all resources must follow our formatting rules:

| Resource Type | Accepted File Formats | Target Metadata & Requirements |
| :--- | :--- | :--- |
| **Whitepapers / eBooks** | `.pdf`, `.epub` | Must have selectable text (OCR complete), defined table of contents, and valid citations. |
| **Technical Courses** | `.mp4`, `.mov`, `.mkv` | HD resolution (720p or 1080p). Maximum 500MB file size. Screen-recording must be clear. |
| **Developer Templates** | `.zip`, `.tar.gz` | Include a structured `README.md`, setup commands, and basic test coverages. |
| **Design Kits** | `.fig`, `.zip` (Figma / Sketch) | Structured layer groups, global color/typography tokens, and responsive layout grids. |

---

## 2. Categorization & Metadata

Accurate categorization ensures your resource is discoverable by scholars and students:

### Topics (Subject Matter)
Select the most relevant topic category:
- **Smart Contracts**: Solidity, Vyper, Rust, ZK circuits, and audit reports.
- **DeFi (Fintech)**: AMM architecture, liquidity models, staking, and tokenomics.
- **UI/UX Design**: dApp wireframes, user flow mapping, and UI components.
- **AI Ethics**: Algorithmic fairness, model audit, and decentralized AI.
- **Big Data / Cybersecurity**: Datasets, cryptography, and network security.

### Difficulty Levels
Select the appropriate level to set realistic learner expectations:
- **Beginner**: Introductory material, no prior blockchain coding knowledge required.
- **Intermediate**: Basic familiarity with Solidity, React, or cryptography required.
- **Advanced**: Complex multi-file integrations, deep zero-knowledge proofs, or low-level optimizations.
- **Expert**: State-of-the-art research papers, mathematical proofs, or production-grade audits.

---

## 3. Access Controls & Token Pricing

You can distribute resources under two pricing models:

### Premium Tier (BTS Monetization)
- Specify a price in **BTS tokens** (e.g., 15 BTS to 200 BTS).
- Users must purchase the resource to download it.
- **Reputation Yield**: Active, highly rated premium papers generate a secondary yield from the Bitstacks network pool.

### Open Access Tier (Public Good)
- Renders with a green **Public Good** badge.
- Free for all Bitstacks users.
- **Reputation Boost**: Free resources build your author reputation score **2.5x faster** than premium listings, unlocking governance voting weight.

---

## 4. Strict Policies & Guidelines

Please adhere strictly to our academic integrity policies. All submissions undergo randomized peer review.

> [!IMPORTANT]
> **Copyright Clearance**
> You must hold the intellectual property or appropriate open-source distribution licenses for all codebases, UI assets, and textbooks submitted.

> [!WARNING]
> **AI Disclosure Policy**
> Any resource created primarily using Generative AI (e.g., text, code, or images) must state **"AI-Generated Content Disclosed"** at the top of the description. Failure to disclose AI usage will result in permanent exclusion from publishing.

---

## 5. Step-by-Step Submission Process

1. **Step 1 (Resource Info)**: Enter Title, Description, and select category, difficulty, and tags.
2. **Step 2 (Content Upload)**: Drag and drop your file or browse to select it. The file type and size will be updated.
3. **Step 3 (Set Pricing)**: Select **Premium** (enter BTS price) or **Open Access**.
4. **Step 4 (Review)**: Double-check all entries, and click **Publish Resource**. Your resource will go live on the D-Library page instantly.
